Wayland Wire Protocol — Architecture Plan

A pure-Zig Wayland client that speaks the wire protocol directly — no libwayland, no xkbcommon, no codegen. The goal: wire.zig becomes the complete Wayland module, linux.zig becomes pure policy, and adding new Wayland interfaces is a one-file change.

Guiding principle: three layers, three sources of truth. interfaces.zig = the SPEC. wire.zig = the WIRE (framing, routing, parsing, dispatching). linux.zig = the POLICY (what events mean to the application). No layer should leak its concerns into another. If sending feels clean but receiving is messy, the architecture is wrong.

Architecture Overview

src/platform/
  linux.zig ← pure policy: what events mean
  linux/
    wayland/
      wire.zig ← protocol mechanics
      interfaces.zig ← spec map (Request/Event unions)

Why three layers — the asymmetry argument

The send side currently looks clean: 
try display.send(.{ .get_registry = .{ .registry = registry_id } });
But the receive side leaks protocol details into linux.zig: 
const T = std.meta.fields(WlDisplay.Event)[@intFromEnum(WlDisplay.Event.err)].type;
const msg = try wire.parse(T, payload);
log.err(...)
That asymmetry IS the smell. If Proxy(Iface) is "the way to talk TO this interface," then it must also be "the way to react FROM it." The fix is making the proxy two-way and giving Connection a routing table so it can dispatch events to the right proxy automatically.

Layer responsibilities

wire.zig owns

  • Unix socket connection lifecycle
  • Wire framing — 8-byte Header, payload, 4-byte alignment
  • Serialization (emit) and deserialization (parse)
  • Object ID allocation (ObjIdAllocator)
  • The routing table — maps obj_id → handler
  • The drain loop — read message, look up handler, dispatch
  • delete_id auto-cleanup of routing entries
  • Type erasure machinery for the routing table (comptime trampoline)

interfaces.zig owns

  • Per-interface namespaces: WlDisplay, WlRegistry, Callback, ...
  • Request: union(enum) — outgoing message variants in spec order
  • Event: union(enum) — incoming message variants in spec order
  • Inline comptime asserts confirming variant index = wire opcode
  • Zero logic — pure spec map

linux.zig owns

  • File-scope state: conn, compositor_id, seat_id, etc.
  • File-scope Proxy instances for interfaces it cares about
  • Per-interface event handler functions (the policy)
  • Decisions about what's fatal vs recoverable
  • The plat.* public API (createWindow, pollNextEvent, etc.)

What to notice in the diagrams:
  • linux.zig only ever sees typed values: WlRegistry.Event, u32, []const u8. Never opcodes, never raw bytes outside of the g.interface string.
  • wire.zig handles all the type erasure (*anyopaque) internally. The trampoline reverses it before linux ever gets called.
  • interfaces.zig types travel both directions but the file itself is never executed — it's pure compile-time information.
  • The OS layer sees only []const u8 bytes. No types cross the kernel boundary.

Step 2 — Wire Primitives + Connection

The byte-level mechanics. Mostly done. Cleanup remaining.

What's done

  • WlFixed packed struct (frac:u8, integer:i24)
  • ObjIdAllocator — next: u32 = 2, alloc() returns + increments
  • wlPad(len) inline fn — (4 - @mod(len, 4)) % 4
  • Header packed struct — obj_id: u32, size_opcode: u32 with size() and opcode() methods
  • Message struct — { header: Header, bytes: []const u8 }
  • emit internal byte builder — walks union tag, writes header + payload + size patch
  • parse internal byte reader — walks struct fields, offset-free signature, starts at @sizeOf(Header) internally
  • Connection heap-allocated singleton
  • Connection.init — env-driven socket path, connects, builds writer/reader, initializes ids
  • Connection.deinit — closes stream, destroys self via stored gpa
  • Connection.sendRaw — derives opcode from activeTag, calls emit, writeAll, flush
  • Connection.nextMessage — peekStruct header, take(size), returns Message

Cleanup pending

  • send_buf: [4098]u8 typo → [4096]u8 on line 55 of wire.zig
  • Remove unused recv_buf field on line 56 (the actual reader buffer is read_buf)
  • Remove debug prints in sendRaw (lines 105-111) — leftover from byte verification
  • nextMessage swallows peekStruct error with a fake header on lines 122-125. Change to try and let it bubble — the policy layer decides what to do.
Stale pointer gotcha (already learned): Connection must be heap-allocated and returned as *Connection. The reason: writer and reader hold @fieldParentPtr back-references to themselves inside the struct. If Connection is returned by value, those references become stale immediately. Heap allocation gives a stable address.

Step 3 — Interfaces Stabilization

Lock down the spec contract that Proxy and routing depend on.

What's done

  • snake_case field names everywhere (delete_id, not deleteId)
  • Comptime asserts inline with each interface — living documentation
  • WlErrors hoisted to top-level (not buried inside WlDisplay)

Outstanding

  • Naming consistency — WLDisplay vs WlRegistry in interfaces.zig. Pick Wl (matches Wayland's own casing). Update import aliases in linux.zig.
  • Add empty Request on Callback for shape consistency: 
    pub const Callback = struct {
    pub const Request = union(enum) {};
    pub const Event = union(enum) { done: ... };
};
  • Defer wl_registry.bind special encoding (interface name + version inline with new_id arg) — handle when first needed
Why the empty Request matters: when Proxy(Iface) references Iface.Request, it has to compile. If Callback omits Request, Proxy(Callback) won't compile. Empty union(enum) {} is zero-cost — the compiler discards it — and keeps every interface uniform.

Step 4 — Two-Way Proxy

The first big architectural move. Make Proxy own both directions of communication. This is what removes the asymmetry described in the architecture overview.

The new Proxy shape

pub fn Proxy(comptime Iface: type) type {
    return struct {
        obj_id: u32,
        conn: *Connection,
        on_event: *const fn (event: Iface.Event) anyerror!void,

        const Self = @This();
        pub const Interface = Iface;

        pub fn send(self: Self, msg: Iface.Request) !void {
            try self.conn.sendRaw(self.obj_id, msg);
        }

        pub fn handle(self: Self, opcode: u16, bytes: []const u8) !void {
            const event = try parseEvent(Iface.Event, opcode, bytes);
            try self.on_event(event);
        }
    };
}

The supporting parseEvent generic

pub fn parseEvent(
    comptime EventUnion: type,
    opcode: u16,
    data: []const u8,
) !EventUnion {
    inline for (@typeInfo(EventUnion).@"union".fields, 0..) |f, i| {
        if (i == opcode) {
            const inner = try parse(f.type, data);
            return @unionInit(EventUnion, f.name, inner);
        }
    }
    return error.UnknownOpcode;
}

What changes for the caller

Sends already worked. Receives now work the same way:

// before: linux.zig dispatches manually
const T = std.meta.fields(WlDisplay.Event)[@intFromEnum(...)].type;
const msg = try wire.parse(T, bytes);

// after: Proxy.handle owns it
try display_proxy.handle(opcode, bytes);  // calls parseEvent + on_event

But this is just the foundation — Step 5 wires up Connection so linux.zig doesn't even call handle manually. Connection's drain loop does it.

Tasks

  • Add parseEvent generic to wire.zig (after parse)
  • Add Proxy(comptime Iface: type) generic with obj_id, conn, on_event fields
  • Add Proxy.send method (forwards to conn.sendRaw)
  • Add Proxy.handle method (parseEvent + on_event)
  • Add pub const Interface = Iface; for callers that want to reach Proxy(X).Interface.Event
on_event is a plain function pointer — no closure capture, no context object. Linux's file-scope vars (compositor_id, etc.) are accessed directly inside the handler fns. The file-as-singleton pattern makes this work without any plumbing.

Step 5 — Connection Owns Routing

The second big architectural move. Move the dispatch table into Connection. Wire becomes the full hub.

Connection gains a routing table

pub const Connection = struct {
    // ... existing fields ...
    handlers: std.AutoHashMap(u32, Handler),

    const Handler = struct {
        ctx: *anyopaque,
        handle_fn: *const fn (
            ctx: *anyopaque,
            opcode: u16,
            bytes: []const u8,
        ) anyerror!void,
    };

    pub fn registerProxy(
        self: *Connection,
        comptime Iface: type,
        proxy: *Proxy(Iface),
    ) !void {
        const trampoline = struct {
            fn handle(ctx: *anyopaque, opcode: u16, bytes: []const u8) !void {
                const p: *Proxy(Iface) = @ptrCast(@alignCast(ctx));
                try p.handle(opcode, bytes);
            }
        };
        try self.handlers.put(proxy.obj_id, .{
            .ctx = proxy,
            .handle_fn = trampoline.handle,
        });
    }

    pub fn unregisterProxy(self: *Connection, obj_id: u32) void {
        _ = self.handlers.remove(obj_id);
    }

    pub fn drain(self: *Connection, stop_obj_id: u32) !void {
        while (true) {
            const m = try self.nextMessage();
            if (self.handlers.get(m.header.obj_id)) |h| {
                try h.handle_fn(h.ctx, m.header.opcode(), m.bytes);
            } else {
                log.warn(.platform, "no handler for obj_id={} opcode={}",
                    .{ m.header.obj_id, m.header.opcode() });
            }
            // wl_display.delete_id auto-removes routing entry
            if (m.header.obj_id == 1 and m.header.opcode() == 1) {
                const deleted = std.mem.readInt(
                    u32,
                    m.bytes[@sizeOf(Header)..][0..4],
                    .little,
                );
                _ = self.handlers.remove(deleted);
            }
            if (m.header.obj_id == stop_obj_id) return;
        }
    }
};

The trampoline pattern explained

The routing table needs to store handlers for any interface type uniformly. But each interface has a different Proxy(Iface) type, and std.AutoHashMap needs a single value type. Solution: type erasure via *anyopaque + a per-interface trampoline function.

The trampoline is synthesized at comptime per call to registerProxy: each call creates a new closure-like struct with a handle fn that casts ctx back to the typed *Proxy(Iface). This is the exact pattern Zig's stdlib uses for vtables (e.g., std.Io.Reader.streamImpl).

Drain auto-cleanup

Wayland's wl_display.delete_id event tells the client "this object is gone, the id can be reused." Wire owns the routing table, so wire should respond to this event by removing the entry. Linux doesn't need to know — the bookkeeping is purely a wire concern.

Special-casing this in drain is correct because wl_display is fixed at obj_id=1 and delete_id is fixed at opcode 1. Both are set by the protocol forever.

Tasks

  • Add handlers: std.AutoHashMap(u32, Handler) field to Connection
  • Initialize handlers in Connection.init with the gpa
  • Free handlers in Connection.deinit (call handlers.deinit())
  • Add internal Handler struct (ctx + fn ptr)
  • Add registerProxy with comptime trampoline synthesis
  • Add unregisterProxy for explicit removal
  • Add drain method with delete_id auto-cleanup and stop_obj_id support
  • Decision: keep nextMessage public for advanced uses, or hide it. Recommendation: keep public — useful for testing and for any caller who wants to bypass routing.
Map deinit order: when Connection.deinit runs, free handlers BEFORE gpa.destroy(self). Otherwise you free the gpa-allocated memory while the map still holds references to its own allocator inside Connection.

Step 6 — linux.zig as Pure Policy

Final shape. linux.zig knows nothing about the protocol — it just registers proxies and writes typed switches on events.

The end-state file

const std = @import("std");
const Allocator = std.mem.Allocator;
const plat = @import("platform.zig");
const wire = @import("linux/wayland/wire.zig");
const wli = @import("linux/wayland/interfaces.zig");
const log = @import("debug").log;

const Event = plat.Event;
const WindowConfig = plat.WindowConfig;
const Capabilities = plat.Capabilities;
const DisplayInfo = plat.DisplayInfo;

var gpa: Allocator = undefined;
var conn: *wire.Connection = undefined;

// known global ids (filled in by onRegistryEvent)
var compositor_id: u32 = 0;
var xdg_wm_base_id: u32 = 0;
var seat_id: u32 = 0;
var compositor_name: u32 = 0;     // for bind
var xdg_wm_base_name: u32 = 0;
var seat_name: u32 = 0;

// proxies (file-scope so handler fns can reach them if needed)
var display_proxy: wire.Proxy(wli.WlDisplay) = undefined;

    xdg_surface_id: u32,

    pub fn swapBuffers(self: *const Window, offset: u32) void { _ = self; _ = offset; }

// ── platform interface ────────────────────────────────────────
pub fn init(p_gpa: Allocator, p_io: std.Io, env: *std.process.Environ.Map) !void {
    gpa = p_gpa;
    conn = try wire.Connection.init(gpa, p_io, env);

    // wl_display is fixed at obj_id=1
    display_proxy = .{
        .obj_id = 1,
        .conn = conn,
        .on_event = onDisplayEvent,
    };
    try conn.registerProxy(wli.WlDisplay, &display_proxy);

    // get a registry id and register a proxy for it
    const registry_id = conn.ids.alloc();
    registry_proxy = .{
        .obj_id = registry_id,
        .conn = conn,
        .on_event = onRegistryEvent,
    };
    try conn.registerProxy(wli.WlRegistry, ®istry_proxy);

    // ask compositor to send us all the globals
    try display_proxy.send(.{ .get_registry = .{ .registry = registry_id } });

    // sync barrier — drain until callback fires
    const cb_id = conn.ids.alloc();
    try display_proxy.send(.{ .sync = .{ .callback = cb_id } });
    try conn.drain(cb_id);

    log.info(.platform, "globals enumerated: compositor={} xdg_wm_base={} seat={}",
        .{ compositor_name, xdg_wm_base_name, seat_name });
}

pub fn deinit() void {
    conn.deinit();
}

// ── event handlers (the policy) ───────────────────────────────
fn onDisplayEvent(event: wli.WlDisplay.Event) !void {
    switch (event) {
        .err => |e| {
            log.err(.platform, "wl_display error: obj={} code={} msg={s}",
                .{ e.object_id, e.code, e.msg });
            return error.WaylandProtocolError;
        },
        .delete_id => {},  // wire handles routing cleanup automatically
    }
}

fn onRegistryEvent(event: wli.WlRegistry.Event) !void {
    switch (event) {
        .global => |g| {
            log.info(.platform, "global: {s} v{}", .{ g.interface, g.version });
            if (std.mem.eql(u8, g.interface, "wl_compositor")) {
                compositor_name = g.name;
            } else if (std.mem.eql(u8, g.interface, "xdg_wm_base")) {
                xdg_wm_base_name = g.name;
            } else if (std.mem.eql(u8, g.interface, "wl_seat")) {
                seat_name = g.name;
            }
        },
        .global_remove => {},
    }
}

Errors bubble cleanly

Per the policy decision: handler returns error → drain returns error → init returns error → app decides what to do.

onDisplayEvent returning error.WaylandProtocolError on the .err variant is the canonical example. Display-level errors are fatal — bubble them. Other handlers might log and continue (e.g., unknown global → warn, keep going). Wire never decides recovery, only reports.

Adding a New Interface (post-completion)

After steps 4-6 land, supporting a new Wayland interface is a four-step pattern with no wire.zig changes:

1. Declare the interface in interfaces.zig

pub const WlCompositor = struct {
    pub const Request = union(enum) {
        create_surface: struct { id: u32 },
        create_region: struct { id: u32 },
    };
    pub const Event = union(enum) {};
    comptime {
        std.debug.assert(@intFromEnum(WlCompositor.Request.create_surface) == 0);
        std.debug.assert(@intFromEnum(WlCompositor.Request.create_region) == 1);
    }
};

2. Add file-scope state in linux.zig

var compositor_id: u32 = 0;
var compositor_proxy: wire.Proxy(wli.WlCompositor) = undefined;

3. Write the handler fn

fn onCompositorEvent(event: wli.WlCompositor.Event) !void {
    switch (event) {
        // empty union — no events for compositor
    }
}

4. Register and use in init

// after registry roundtrip has populated compositor_name:
compositor_id = conn.ids.alloc();
try registry_proxy.send(.{ .bind = .{
    .name = compositor_name,
    .new_id = compositor_id,
} });
compositor_proxy = .{
    .obj_id = compositor_id,
    .conn = conn,
    .on_event = onCompositorEvent,
};
try conn.registerProxy(wli.WlCompositor, &compositor_proxy);

wire.zig is never touched. That's the entire point of the architecture.

Out of Scope (deferred, not forgotten)

  • wl_registry.bind special encoding — the new_id arg on the wire carries the interface name string and version inline. Every other request uses bare u32 new_ids. Special-case in emit when implementing window creation.
  • SCM_RIGHTS file descriptor passing — wl_keyboard.keymap sends a memory-mapped fd via Unix socket ancillary data, not in the message payload. Requires sendmsg with cmsg. Handle when keyboard input matters.
  • Non-blocking poll — current nextMessage blocks. pollNextEvent needs a non-blocking variant of drain. Add when game-loop integration starts.
  • Send batching — multiple requests per frame could share one flush call. Wire-internal optimization, never visible to linux.zig.
  • Comptime-synthesized convenience methods — auto-generate display.getRegistry(id) from the union variants. Worth doing if surface area grows past ~20 requests. For now, the union literal call site is acceptable verbosity.
  • Multi-window support — currently the architecture assumes one window. When multiple windows are needed, surface_id → Window routing has to live somewhere — probably linux.zig owns a hashmap of surface_id → *Window. Wire stays single-connection.
  • Generated zwayland — eventual semester-project goal: parse wayland.xml at comptime and generate interfaces.zig automatically. For this engine, hand-writing 6-8 interfaces is faster than building the generator.
