Macro Overview

Main Loop

src/tools/zixelart/main.zig — the frame-by-frame heartbeat

  1. app.beginFrame() — poll OS events, clear input frame states
  2. Keyboard input — Esc: quit | C: clear canvas | Cmd+Z: undo | Cmd+Shift+Z: redo
  3. screenToCanvas() — convert mouse screen position to canvas grid coords, or null if outside
  4. Mouse dispatch — left button pressed → canvas.onMouseDown() | dragged → onMouseDrag() | released → onMouseUp()
  5. Render canvas — iterate canvas.pixels[], draw each PixelCell via renderer.drawGeometry()
  6. UI update — ui_layer.update() with mouse state
  7. Poll UI — check toolbar tool selection, palette color selection, RGB/HSV slider changes
  8. Sync state — update ZixelState.active_tool, active_color, sync sliders bidirectionally
  9. UI render + app.endFrame() — draw widgets, present frame

Tool Implementations

src/tools/zixelart/tools/ — each tool exports fns() → ToolFns

PencilTool
begin: set pixel color, record PixelChange
update: skip if pixel already in changes (no duplicates per stroke), else begin()
commit: duplicate changes → ToolCommand{ kind: .pencil }
preview: no-op   cancel: canvas.cancel() + clear changes
EraserTool
begin: set pixel to canvas.blank_color (ignores color param), record change
update: skip duplicates, else begin()
commit: ToolCommand{ kind: .erase }
preview: no-op   cancel: canvas.cancel()
FillTool
begin: flood-fill from (x, y)
  Stack-based BFS, capacity 1024. Check target_color != fill color. Push 4 neighbors.
update: no-op (fill only on click)
commit: ToolCommand{ kind: .fill }
preview: no-op   cancel: canvas.cancel()
LineTool
begin: set start pixel, record change
update: restore all previous pixels → clear changes → Bresenham line from start to current
  isize math, dx/dy/sx/sy, error accumulation, records each pixel
commit: ToolCommand{ kind: .line }
preview: no-op   cancel: canvas.cancel()
PickerTool
begin / update / preview / cancel: all no-op
commit: returns null (no canvas modification)
Color picking is handled in main.zig by reading pixel color into state
ToolDispatcher PixelChange / ToolCommand Canvas (pixel data)

Commands & History

src/tools/zixelart/commands.zig + CommandHistory.zig

PixelChange
x: usize   y: usize   old_color: ?Color   new_color: ?Color
ToolCommand
kind: Tool // which tool produced this
pixels: []PixelChange // all changes in one stroke
undo(self, canvas) // restore old_color for each pixel

CommandHistory(T)
Generic over command type (T = ToolCommand)
cmds: ArrayList(T)   cursor: usize // position in stack
Methods push(cmd) // truncates redo stack at cursor, appends
undo() → ?T // decrement cursor, return cmd
redo() → ?T // return cmd at cursor, increment
canUndo() / canRedo() → bool

Undo/Redo Flow

  1. Tool stroke completes → commit() returns ToolCommand with all PixelChanges
  2. history.push(cmd) — appends to stack, truncates any redo future
  3. Cmd+Z → history.undo() → cmd.undo(canvas) restores old_color per pixel
  4. Cmd+Shift+Z → history.redo() → cmd.redo(canvas) restores new_color per pixel
ToolDispatcher (produces commands) Canvas (owns history) Main loop (Cmd+Z / Cmd+Shift+Z)

Data Flow

Tool Execution

  1. Mouse pressed on canvas → canvas.onMouseDown(active_tool, x, y, color)
  2. Dispatcher calls tool's beginFn(canvas, x, y, color)
  3. Tool modifies canvas.pixels[].color, appends to canvas.changes
  4. Mouse dragged → onMouseDrag → tool's updateFn (may skip duplicates)
  5. Mouse released → onMouseUp → tool's commitFn → returns ?ToolCommand
  6. ToolCommand pushed to canvas.history

Color Selection

  1. Palette click → main.zig polls color_ids[] from UILayer → sets state.active_color → syncSlidersToColor()
  2. RGB slider change → build Color.initRgba(r*255, g*255, b*255, 255) → sets state.active_color → syncHsvSliders()
  3. HSV slider change → build Color.initHsva(h*360, s, v, 1) → sets state.active_color → syncRgbSliders()

Render Pipeline

  1. app.beginFrame() — poll events, start GPU frame
  2. Iterate canvas.pixels[] → renderer.drawGeometry(cell.shape, null, cell.color, BLACK, 1.0, ctx)
  3. ui_layer.render() — all widget trees drawn
  4. app.endFrame() — flush to GPU, present
