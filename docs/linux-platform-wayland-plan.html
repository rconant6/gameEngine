Linux Platform Layer — Wayland + Vulkan Handoff

Clean-slate rebuild of linux.zig and the entire platform/linux/wayland/ layer. Uses libwayland for the connection, surface objects, and event loop — but replaces all C listener structs with a comptime Proxy(T) dispatcher. Ends at the renderer handoff: getNativeWindowHandle() returning a real wl_display* and wl_surface* for Vulkan.

Why this architecture? libwayland is kept for three things only: connect the socket, create surfaces Vulkan can use, and run the dispatch loop. Everything above that — typed events, single handler per interface, comptime decode — is your own layer, identical in shape to the old wire approach. wl_proxy_add_dispatcher is the seam between them.

Phase 2

Proxy Dispatcher new

The core of the new approach. Replaces the event-handling half of wire.zig. libwayland reads the socket and hands you a wl_argument[] array — this layer decodes it into a typed Zig event union and calls your handler.

Proxy(T) struct

  • Generic over interface type T (e.g. WlSeat, XdgSurface)
  • Field: ctx: *anyopaque — pointer to WaylandState or WindowState
  • Field: on_event: *const fn(T.Event, *anyopaque) anyerror!void
  • Method: register(proxy: *c.wl_proxy) void — calls c.wl_proxy_add_dispatcher
Memory: The Proxy(T) struct must live in stable heap memory — libwayland stores a pointer to it as the implementation argument. Store proxy instances directly inside WaylandState or WindowState, which are heap-allocated. Never store on the stack.

C dispatch function

  • One static callconv(.c) function per Proxy(T) instantiation
  • Signature: (impl: ?*const anyopaque, target: ?*anyopaque, opcode: u32, msg: ?*const c.wl_message, args: ?[*]c.wl_argument) c_int
  • Cast impl → *Proxy(T), call decodeArgs, call on_event
  • Return 0 on success, -1 on error

decodeArgs — comptime decoder

Same structure as old parseEvent in wire.zig, just reading from wl_argument[] instead of raw bytes.

  • inline for over T.Event union fields, match opcode index
  • inline for over that variant's struct fields — map Zig type → wl_argument field:
  • u32 → args[i].u
  • i32 → args[i].i
  • []const u8 → std.mem.span(args[i].s)
  • WlFixed → @bitCast(args[i].u)
  • WlArray → read from args[i].a (wl_array: .size, .data)
  • i32 fd (keyboard keymap) → args[i].h
  • Empty struct (e.g. close, frame) → no args consumed, construct .{}
Pointer cast: wl_proxy_add_dispatcher takes *c.wl_proxy, but libwayland hands you typed pointers (*c.wl_registry, *c.wl_seat, etc.). All Wayland objects ARE wl_proxy internally. Use @ptrCast — this is intentional in libwayland's design, not a hack.
Phase 3

State port / adapt

Port _oldWireWayland/state.zig. Main difference: BoundObject(T) now holds a C handle alongside the Proxy.

  • version: u32 — filled during registry scan
  • proxy: Proxy(T) — filled after bind + register

WaylandState

  • display: *c.wl_display
  • registry: *c.wl_registry
  • compositor: ?*c.wl_compositor

  • registry_proxy: Proxy(WlRegistry)
  • compositor_info: BoundObject(WlCompositor)
  • seat_info: BoundObject(WlSeat)
  • xdg_wm_base_info: BoundObject(XdgWmBase)
  • output_info_bound: BoundObject(WlOutput)
  • has_pointer: bool

Phase 4

Handlers port / adapt

Port _oldWireWayland/handlers.zig. The typed switch logic is identical. The only adaptation: responses to compositor events (ping, ack) now call the C API instead of sending over the wire.

State-level handlers (ctx = *WaylandState)

  • onRegistryEvent — fills xxx_info.name + xxx_info.version for each known interface string
  • onCompositorEvent — empty (no compositor events)
  • onSeatEvent — sets has_pointer / has_keyboard from capability bitmask; logs seat name
  • onOutputEvent — fills output_info (geometry, mode, scale)
  • onXdgWmBaseEvent — ping → call c.xdg_wm_base_pong(state.xdg_wm_base, serial)
  • onPointerEvent — translate to engine Event and push to window event queue
  • onKeyboardEvent — translate to engine Event and push to window event queue

Window-level handlers (ctx = *WindowState)

  • onXdgSurfaceEvent — configure → store serial, call c.xdg_surface_ack_configure(win.xdg_surface, serial), set configured = true
  • onXdgToplevelEvent:
    • configure → update width / height if non-zero
    • close → set should_close = true
    • configure_bounds → log or ignore
    • wm_capabilities → log or ignore
Event routing: Pointer and keyboard handlers need access to the active window's event queue. Pass the window pointer via ctx, or store a pointer to the current focused window in WaylandState.
Phase 5

Platform API — linux.zig rewrite

The top layer. Orchestrates init, window creation, the event loop, and exposes the platform API that the rest of the engine calls. This file is pure policy — no Wayland wire knowledge lives here.

init()

  • wl_display_connect(null) → store in state.display
  • wl_display_get_registry(display) → store in state.registry
  • Create state.registry_proxy with ctx = state, on_event = handlers.onRegistryEvent
  • registry_proxy.register(@ptrCast(state.registry))
  • wl_display_roundtrip — fires registry globals, fills xxx_info.name/version
  • Bind compositor: wl_registry_bind → state.compositor, create + register compositor_info.proxy
  • Bind xdg_wm_base: same pattern → state.xdg_wm_base, register proxy with onXdgWmBaseEvent
  • Bind seat: same → state.seat, register proxy with onSeatEvent
  • Bind output: same → state.output, register proxy with onOutputEvent
  • wl_display_roundtrip — fires seat capabilities, output mode/scale

deinit()

  • Destroy objects in reverse creation order
  • wl_display_disconnect(state.display) last

createWindow(config)

  • wl_compositor_create_surface(state.compositor) → win.surface
  • xdg_wm_base_get_xdg_surface(state.xdg_wm_base, surface) → win.xdg_surface
  • xdg_surface_get_toplevel(xdg_surface) → win.xdg_toplevel
  • Create + register win.xdg_surface_proxy with onXdgSurfaceEvent, ctx = win
  • Create + register win.xdg_toplevel_proxy with onXdgToplevelEvent, ctx = win
  • xdg_toplevel_set_title
  • wl_surface_commit
  • wl_display_roundtrip — compositor sends configure, handler acks it
  • Return *Window (wraps WindowState + exposes platform Window API)

Event Loop

  • pollNextEvent() — wl_display_dispatch_pending then pop from window event queue
  • waitEvent() — wl_display_dispatch (blocks) then pop

Platform API stubs + implementations

  • shouldClose(window) → win.should_close
  • getWindowScaleFactor(window) → state.output_info.scale
  • getDisplays(allocator) → populate from state.output_info
  • setMouseCursorVisible / setMouseCursorLocked — stub ok for now
  • swapBuffers — stub (Vulkan owns presentation)
  • getClipboardText / setClipboardText — return error.NotImplemented
  • getCapabilities() — return has_vulkan: true

Renderer Handoff Point

This is where the platform layer ends and the renderer begins. getNativeWindowHandle() must return a heap-allocated struct containing two pointers Vulkan needs:

  • WindowHandle.display: *anyopaque — cast from *c.wl_display
  • WindowHandle.surface: *anyopaque — cast from *c.wl_surface
  • Caller allocates, caller destroys — document this in the function signature
  • Vulkan side uses these as VkWaylandSurfaceCreateInfoKHR.display and .surface

Once this function returns valid pointers and the window is configured, the renderer can create a VkSurfaceKHR and take over. The platform layer's job is done.
