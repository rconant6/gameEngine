Zixelart UI Development Guide

Goal: Build a complete pixel art editor UI with tool bar, canvas, palette, shades panel, preview, and info bar using the existing rendering system with hardcoded layout values.

Current State: Canvas rendering works, palette data structure exists, need to add screen-space text and wire up the full UI layout.

End State: Functional editor with clickable tools, color selection, and real-time info display.

Phase 0: Prerequisites

Add Screen-Space Text Rendering

Before building the UI, you need drawText to work in screen coordinates (like RectangleScreen).

Task 0.1: Add drawTextScreen to text.zig

File: src/renderer/text.zig

Goal: Create a screen-space variant of drawText


Steps:

  1. Copy the existing drawText function signature
  2. Change position: WorldPoint to position: ScreenPoint
  3. Name it drawTextScreen
  4. Inside, use Triangle(ScreenPoint) instead of Triangle(WorldPoint)
  5. The glyph building logic stays the same - just different coordinate type

Function Signature:

pub fn drawTextScreen(renderer, font, text, position: ScreenPoint, scale, color, ctx) void

Task 0.2: Expose drawTextScreen in Renderer

File: src/renderer/renderer.zig

Goal: Add a drawTextScreen method that calls the new function


Steps:

  1. Find the existing drawText method in Renderer struct (~line 133)
  2. Add a parallel method drawTextScreen below it
  3. Call text_module.drawTextScreen(...)
Checkpoint: You can now render text at screen pixel coordinates like {.x = 100, .y = 50}

Phase 1: Layout Structure

Create UI Layout File

Define all the layout constants and region boundaries in one place.

src/tools/zixelart/Layout.zig

Purpose: Single source of truth for all UI positions and sizes


Contents:

  • Screen dimension constants
  • Region structs with x, y, width, height
  • Helper functions: getToolbarRegion(), getPaletteRegion(), etc.
  • Hit-test helpers: isInToolbar(x, y), isInPalette(x, y), etc.

Regions to Define:

  • toolbar - x: 0, y: 40, w: 80, h: 1048
  • canvas - x: 326, y: 40, w: 1048, h: 1048
  • palette - x: 1620, y: 40+648, w: 300, h: 400
  • shades - x: 1620, y: 40+348, w: 300, h: 300
  • preview - x: 1620, y: 40, w: 300, h: 348
  • info_bar - x: 0, y: 0, w: 1920, h: 40

Note: Y coordinates are from bottom (0) to top (1088) in your coordinate system. Adjust accordingly. The info bar is at the BOTTOM (y: 0), and content grows upward.

Render Steps (for each tool):

Task 3.1: Implement handleClick

Logic:

  1. Check if x is within toolbar bounds (0 to 80)
  2. Check if y is within content area (40 to 1088)
  3. Calculate which tool index: (CONTENT_HEIGHT - y) / TOOL_HEIGHT
  4. If index < 5, return tools[index]
  5. Otherwise return null
Task 3.2: Wire Up in main.zig

On Mouse Click:

  1. First check toolbar.handleClick(screen_x, screen_y)
  2. If returns a tool, update state.active_tool
  3. Otherwise, continue with existing canvas click logic
Checkpoint: You see 5 tool labels on the left. Clicking them updates the info bar to show the selected tool. The active tool is highlighted.

Phase 4: Color Palette

Render 64-Color Grid with Selection

8x8 grid of color swatches in the right panel.

Task 4.1: Update Palette.zig

File: src/tools/zixelart/Palette.zig


Replace Current Implementation:

The current Palette uses generated gradients. Replace with the Crayola 64 or a curated set.


New Struct Fields:

  • colors: [64]Color - the palette colors
  • names: [64][]const u8 - color names for info bar
  • region: Layout.Region - bounds from Layout
  • swatch_size: usize - calculated: region.width / 8

Init Options (pick one):

  • Option A: Use Crayola 64 - Colors.CRAYOLA_*
  • Option B: Use Library queries - Library.getHueToneSat()
  • Option C: Hardcode a pixel-art friendly palette
Task 4.2: Add Palette Rendering

Function: render(renderer, selected_index, ctx)

  1. Draw panel background (slightly lighter than app background)
  2. For each color (0..64):
    • Calculate grid position: col = i % 8, row = i / 8
    • Calculate screen position from region + grid position
    • Draw filled rectangle with color
    • If selected, draw thicker/brighter border
    • Otherwise draw thin dark border
Task 4.3: Add Click Handling

Function: handleClick(screen_x, screen_y) ?usize

  1. Check if click is within palette region
  2. Calculate local coordinates: local_x = screen_x - region.x
  3. Calculate grid cell: col = local_x / swatch_size
  4. Calculate index: row * 8 + col
  5. Return index if valid (< 64), null otherwise
Task 4.4: Update ZixelState

Add Field:

  • selected_palette_index: usize = 0

Update on Palette Click:

  1. Set state.selected_palette_index
  2. Set state.active_color = palette.colors[index]
  3. Set state.active_color_name = palette.names[index]
Checkpoint: You see an 8x8 color grid on the right. Clicking a color selects it (highlighted border), updates the info bar, and drawing uses that color.

Phase 5: Tints & Shades Panel

Auto-Generated Tints and Shades from Active Color

10 vertical swatches: 5 tints (lighter) + 5 shades (darker) of the current color.

src/tools/zixelart/TintShadePanel.zig

Purpose: Show lighter and darker variations for easy shading


Struct Fields:

  • region: Layout.Region
  • swatch_height: usize - region.height / 10

Functions:

FunctionPurpose
getTints(base_color) [5]ColorGenerate 5 tints using gen.tints()
getShades(base_color) [5]ColorGenerate 5 shades using gen.shades()
render(renderer, base_color, ctx)Draw 10 vertical swatches
handleClick(screen_x, screen_y, base_color) ?ColorReturn clicked tint/shade color

Color Generation:

  • gen.tints(base_color, 5) - ramp from base toward white (lighter)
  • gen.shades(base_color, 5) - ramp from base toward black (darker)

Layout (top to bottom):

  1. Tint 5 (lightest)
  2. Tint 4
  3. Tint 3
  4. Tint 2
  5. Tint 1
  6. Shade 1
  7. Shade 2
  8. Shade 3
  9. Shade 4
  10. Shade 5 (darkest)
Task 5.1: Implement Rendering

Render Steps:

  1. Call getTints(state.active_color) to get 5 tint colors
  2. Call getShades(state.active_color) to get 5 shade colors
  3. For each of the 10 swatches (0..10):
    • Use tints[4-i] for i=0..4, shades[i-5] for i=5..9
    • Calculate Y: region.y + region.height - (i + 1) * swatch_height
    • Draw filled rectangle spanning full width
    • Draw subtle border between swatches
Task 5.2: Wire Up Click Handling

On Tint/Shade Click:

  1. Get the clicked tint or shade color
  2. Set as state.active_color
  3. Set state.active_color_name = "Custom" (or snap to library name)
  4. Do NOT change selected_palette_index - tints/shades are temporary
Checkpoint: When you select a palette color, 5 tints (top) and 5 shades (bottom) appear. Clicking any swatch lets you draw with it. The palette selection stays the same.

Phase 6: Preview Panel

1:1 Scale Canvas Preview

Shows what your pixel art looks like at actual 64x64 pixel size.

src/tools/zixelart/Preview.zig

Purpose: Render a tiny version of the canvas at true pixel scale


Struct Fields:

  • region: Layout.Region

Functions:

FunctionPurpose
render(renderer, canvas, ctx)Draw 64x64 preview centered in region

Render Steps:

  1. Draw panel background
  2. Calculate centered position for 64x64 preview:
    • preview_x = region.x + (region.width - 64) / 2
    • preview_y = region.y + (region.height - 64) / 2
  3. Draw border around preview area
  4. For each canvas pixel:
    • Draw 1x1 screen pixel at (preview_x + px, preview_y + py)
    • Color from canvas.pixels[py * 64 + px].color

Performance Note: Drawing 4096 individual 1x1 rectangles might be slow. If so, consider:
  • Batching into a texture (future optimization)
  • Only updating when canvas changes
  • Drawing at 2x scale (128x128) for now
Checkpoint: You see a tiny 64x64 pixel preview of your art in the bottom-right. It updates in real-time as you draw.

Quick Reference: Coordinate System

Remember: Your coordinate system has Y=0 at the BOTTOM, Y=1088 at the TOP.

  • Info bar is at Y: 0-40 (bottom)
  • Content area is Y: 40-1088 (above info bar)
  • When stacking UI elements from top, calculate: SCREEN_HEIGHT - element_top - element_height
